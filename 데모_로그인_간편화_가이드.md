# 데모 로그인 간편화 가이드

---

## 현재 상황 분석

### 관리자 로그인 (`/admin/login`)
- 비밀번호 하나만 입력 (`admin1234`)
- 이미 페이지 하단에 힌트 텍스트 있음
- **문제**: 그래도 직접 타이핑해야 함 → 고객이 귀찮아서 안 해봄

### 회원 로그인 (`/login`)
- 전화번호 + 비밀번호 입력 필요
- **문제**: 전화번호를 뭘 넣어야 할지 모름 → 고객이 포기함
- **핵심 문제**: 실제 DB에 계정이 있어야 로그인 가능

---

## 해결 전략

```
관리자: "데모 로그인" 버튼 → admin1234 자동 입력 후 바로 로그인

회원:   "데모 체험 시작" 버튼 → 고정 데모 계정으로 회원가입 시도 후 자동 로그인
        (이미 가입된 경우 바로 로그인으로 fallback)
```

데모 회원 계정 정보 (고정값):
```
이름:     데모 회원
전화번호: 010-0000-0001
비밀번호: demo1234
```

---

## 구현 방법 2가지

### 방법 A — 프론트엔드만 수정 (권장 ✅)

백엔드 손대지 않고 프론트에서만 처리.

**회원 로그인 페이지에 버튼 추가:**
1. "데모 체험 시작" 버튼 클릭
2. 자동으로 `memberRegister(데모계정)` 호출
3. 이미 가입된 경우(`409`) → 자동으로 `memberLogin(데모계정)` 호출
4. 로그인 성공 → 대시보드로 이동

**장점**: 백엔드 수정 불필요, 구현 간단
**단점**: 최초 클릭 시 회원가입 API 1회 호출됨 (이후는 로그인만)

---

### 방법 B — 백엔드에 demo 전용 엔드포인트 추가

`POST /api/v2/stores/:storeId/member/demo-login` → 항상 데모 토큰 발급

**장점**: 깔끔
**단점**: 백엔드 수정 필요, 실제 서비스 배포 시 보안 고려 필요

---

### 프롬프트 1 — 관리자 로그인 1-click 버튼

```
frontend/src/views/AdminLogin.vue 파일을 열어줘.

로그인 버튼 위에 "데모 로그인 (1-click)" 버튼을 추가해줘.

동작:
- 클릭 시 password.value를 'admin1234'로 세팅하고 onLogin() 바로 호출
- 버튼 스타일: 현재 로그인 버튼보다 덜 강조되도록
  배경: #f1f5f9 (연한 회색), 텍스트: #475569, 테두리: 1px solid #e2e8f0
  hover 시 배경: #e2e8f0
- 버튼 텍스트: "🎯 데모 계정으로 바로 로그인"
- 기존 비밀번호 힌트 텍스트(페이지 하단 p 태그)는 제거해도 됨
```

---

### 프롬프트 2 — 회원 로그인 데모 버튼

```
frontend/src/views/MemberLoginPage.vue 파일을 열어줘.

로그인/회원가입 카드 위에 "데모 체험 버튼"을 추가해줘.

데모 계정 정보 (코드에 상수로 선언):
  DEMO_NAME = '데모 회원'
  DEMO_PHONE = '01000000001'  (하이픈 없는 숫자)
  DEMO_PASSWORD = 'demo1234'

버튼 클릭 시 동작 (async 함수 demoLogin):
  1. 로딩 상태 시작
  2. memberRegister(STORE_ID, { name: DEMO_NAME, phone: DEMO_PHONE, password: DEMO_PASSWORD }) 호출
  3. 성공 시 → setSession(result.token, ...) 후 /dashboard 이동
  4. 409 에러(이미 가입) 시 → memberLogin(STORE_ID, { phone: DEMO_PHONE, password: DEMO_PASSWORD }) 호출 후 세션 저장 후 이동
  5. 다른 에러 시 → errorMsg에 표시

버튼 위치: 탭(로그인/회원가입) 위에 배치
버튼 스타일:
  - 배경: #0f172a (거의 검정), 텍스트: 흰색
  - 아이콘: 🎯 이모지
  - 텍스트: "데모 계정으로 체험하기"
  - 부제목 (작은 글씨): "전화: 010-0000-0001 / PW: demo1234"
  - border-radius: 12px, padding: 14px
  - 아래에 구분선("또는") 추가 후 기존 탭으로 이어지게

기존 로그인 폼은 그대로 유지.
```

## 데모 시연 동선 (고객에게 보여줄 순서)

```
1. 메인 홈페이지 (/ ) — 랜딩 페이지 전체 스크롤
   └ 히어로 슬라이더, 섹션들, 스크롤 애니메이션 확인

2. 트레이너 페이지 (/trainers) — 트레이너 목록 및 상세

3. 패스 구매 (/passes) — 상품 목록 UI

4. 예약 (/reserve) — 날짜·시간·트레이너 선택 UI

5. 회원 로그인 (/login)
   └ "데모 계정으로 체험하기" 버튼 클릭
   └ 자동 로그인 → 대시보드 진입

6. 회원 대시보드 (/dashboard)
   └ 내 예약, 패스 현황, 알림 확인

7. 관리자 로그인 (/admin/login)
   └ "데모 계정으로 바로 로그인" 클릭

8. 관리자 대시보드 (/admin)
   └ 예약 관리, 회원 목록, 통계 등 순서대로 시연
```

---

## 작업 순서

```
1단계: 프롬프트 1 → 관리자 1-click 로그인 (5분)
2단계: 프롬프트 2 → 회원 데모 버튼 (10분)
3단계: 프롬프트 3 → 안내 박스 추가 (5분)
4단계: 브라우저에서 동선 전체 테스트
5단계: (선택) 프롬프트 4 → 대시보드 데모 배너
6단계: (선택) 프롬프트 5 → 헤더 로그인 버튼 강조
```

---

## 주의사항

- 회원 데모 버튼의 `DEMO_PHONE = '01000000001'`은 DB에 처음 클릭 시 자동 가입됨
- 데모 데이터(예약, 패스 등)는 수동으로 관리자에서 몇 개 미리 만들어 두면 더 좋음
- 실제 클라이언트 납품 코드에는 이 데모 버튼 제거 필요
