## 2. 서비스 배경 및 목적

### 배경

- 소규모 PT 스튜디오는 카카오톡·전화로 예약을 받아 수기 관리하는 경우가 대부분
- 회원권(패스) 잔여 세션을 엑셀·수첩으로 관리해 오류 발생
- 재방문을 유도할 수단이 없어 이탈 회원에 대한 대응이 불가
- 결제는 계좌이체·현금으로만 처리해 정산 관리가 번거로움
- 홈페이지 콘텐츠(이미지·문구) 수정에 개발자 개입이 필요해 운영 비효율 발생

### 목적

- **예약 자동화**: 24시간 온라인 예약으로 전화 응대 부담 제거
- **패스 디지털 관리**: 세션 자동 차감, 잔여 세션 실시간 확인
- **이탈 방지 알림**: 비활동·잔여 세션 경고·재구매 안내 자동 발송
- **온라인 결제**: 토스페이먼츠 연동으로 비대면 패스 구매 지원
- **운영 통계**: 월별 예약·완료율·신규회원·트레이너별 실적 지표 제공
- **자체 CMS**: 홈페이지 이미지·문구·프로그램 카드를 관리자 페이지에서 직접 수정
- **회원 계정**: 전화번호 + 비밀번호 로그인으로 개인 PT 현황 관리

## 3. 타겟 사용자

### 회원 (End User)

- 해당 PT 스튜디오에 등록된 회원
- 전화번호 + 비밀번호로 회원가입·로그인
- 로그인 후 내 PT 현황·예약 이력·패스 상태·운동 달력 확인
- 모바일에서 주로 접근

### 일반 방문자

- 스튜디오를 처음 방문하는 잠재 고객
- 회원가입 없이 홈페이지 조회, 트레이너 소개 확인
- 무료 상담 문의 신청 (이름·연락처·문의 내용)

### 관리자 (Admin)

- 스튜디오 원장 또는 담당 트레이너
- JWT 비밀번호 인증으로 관리 페이지 접근
- 예약·패스·알림·통계·스토어 설정·회원 관리·문의 전반 관리

## 5. 정보 구조 (IA)

```
PT 스튜디오 예약 시스템
│
├── 홈 (/)                              ← 히어로·소개·프로그램 카드·연락처
```

## 6. 사용자 흐름 (User Flow)

### 6.1 회원가입 흐름

```
/login 접속 → 회원가입 탭 선택
  └─▶ 이름 + 전화번호 + 비밀번호(4자↑) + 비밀번호 확인 입력
        └─▶ 가입 요청
              ├─▶ [신규] USER 레코드 생성 + 비밀번호 scrypt 해싱 저장
              ├─▶ [기존 번호, 비밀번호 없음] 비밀번호 추가 등록
              └─▶ JWT 발급 (30일) → memberToken localStorage 저장
                    └─▶ /dashboard 이동
```

### 6.2 로그인 흐름

```
/login 접속 → 로그인 탭 선택
  └─▶ 전화번호 + 비밀번호 입력
        └─▶ PhoneIndex로 사용자 조회
              ├─▶ [성공] JWT 발급 → memberToken 저장 → redirect
              └─▶ [실패] 오류 메시지 (번호 미등록 / 비밀번호 불일치)
```

### 6.3 예약 흐름 (회원)

```
홈 접속 → PT 예약하기 클릭
  └─▶ 전화번호 입력 → 회원 조회
        ├─▶ [기존 회원] 이름 배지 표시
        └─▶ [신규] 이름 입력 → 등록
              └─▶ 날짜 → 시간 → 트레이너(선택) → PT유형 → 패스(선택)
                    └─▶ 예약 제출 → 완료 화면 + SESSION_REMINDER 알림
```

### 6.4 무료 상담 흐름

```
홈 히어로 섹션 → "무료 상담" 버튼 클릭
  └─▶ 상담 모달 (이름·연락처·문의 내용)
        └─▶ 제출 → DynamoDB INQUIRY# 레코드 저장
              └─▶ 성공 토스트 메시지

관리자 /admin/inquiries
  └─▶ 문의 목록 (미확인 강조)
        └─▶ 클릭 → 상세 패널 (자동 읽음 처리)
              └─▶ 확인 완료 처리 / 전화 버튼
```

### 6.5 내 PT 대시보드 흐름

```
헤더 "내 PT·예약" 클릭 or /dashboard 직접 접속
  ├─▶ [비로그인] 로그인 유도 화면 → /login?redirect=/dashboard
  └─▶ [로그인] memberToken 검증
        └─▶ 패스 현황·예약 통계·이번달 운동 횟수·스트릭
              └─▶ 운동 달력·예약 이력·다가오는 예약 표시
```

### 6.6 패스 구매 흐름

```
PT 패스 구매 (/passes) → 전화번호 확인
  └─▶ 상품 선택 → 토스페이먼츠 결제
        └─▶ /payment/success → confirm API → 패스 자동 발급
```

### 6.7 관리자 통계 흐름

```
/admin/stats 접속 (관리자 인증 필요)
  └─▶ 월 선택 ← →
        └─▶ KPI 카드 (총 예약·완료·전체 회원·활성 패스)
              └─▶ 상태 분포 바·6개월 스택 트렌드
                    └─▶ 요일별·시간대별 분포 차트
                          └─▶ 트레이너별 실적 테이블
```

## 7. 페이지별 기능 정의

### 7.1 홈 (`/`)

| 섹션 | 내용 | 관리 |
|------|------|------|
| 히어로 배너 | 배경 이미지 + 타이틀 + 서브 문구 + CTA (무료상담·PT예약) | 스토어 설정 |
| 소개 섹션 1 | 이미지 + 타이틀 + 본문 | 스토어 설정 |
| 소개 섹션 2 | 이미지 + 타이틀 + 본문 (반전 레이아웃) | 스토어 설정 |
| 프로그램 카드 | 최대 6개 카드 (이미지·제목·설명) | 스토어 설정 |
| 연락처 / 지도 | 스튜디오 주소·전화번호·카카오 채널 | 스토어 설정 |

### 7.2 로그인 · 회원가입 (`/login`)

| 탭 | 입력 | 처리 |
|----|------|------|
| 로그인 | 전화번호 + 비밀번호 | PhoneIndex 조회 → scrypt 검증 → JWT 발급 |
| 회원가입 | 이름 + 전화번호 + 비밀번호(4자↑) + 확인 | USER 생성 / 기존 계정에 비밀번호 추가 |

- 비밀번호 표시 토글
- 로그인 성공 후 `redirect` 쿼리 파라미터 경로로 이동

### 7.3 트레이너 목록 / 상세 (`/trainers`, `/trainers/:id`)

| 요소 | 내용 |
|------|------|
| 트레이너 카드 | 프로필 이미지, 이름, 전문 분야, 경력, 소개 |
| 상세 | 전문 분야 태그, 자격증, 프로그램, 가격 |
| CTA | "이 트레이너로 예약하기" |

### 7.4 PT 예약 (`/reserve`)

| 단계 | 내용 |
|------|------|
| 1 | 전화번호 입력 → 회원 조회 (미등록 시 이름 입력) |
| 2 | 날짜 선택 (캘린더) |
| 3 | 시간 선택 (예약된 슬롯 자동 제외) |
| 4 | 트레이너 선택 (선택사항, "무관" 포함) |
| 5 | PT 유형 선택 (30분 / 60분) |
| 6 | 패스 선택 (활성 패스 자동 표시) |
| 7 | 예약 요약 확인 → 제출 |

### 7.5 내 PT · 예약 대시보드 (`/dashboard`)

| 섹션 | 내용 |
|------|------|
| 비로그인 | "로그인 / 회원가입" 버튼 표시 |
| KPI 카드 | 이번 달 운동·전체 완료·연속 주·완료율 |
| 패스 현황 | 활성 패스 카드 (잔여 세션·만료일·프로그레스 바) |
| 운동 달력 | 월별 캘린더, 완료(초록)·예약(파랑)·노쇼(빨강) 색상 표시 |
| 다가오는 예약 | 향후 예약 최대 5건 |
| 예약 이력 | 월별 그룹핑, 전체 이력 |

### 7.6 PT 패스 구매 (`/passes`)

| 상품 | 세션 | 가격 | 유효기간 |
|------|------|------|----------|
| PT 10회권 | 10회 | 500,000원 | 90일 |
| PT 20회권 | 20회 | 900,000원 | 180일 |
| PT 30회권 | 30회 | 1,200,000원 | 180일 |

### 7.7 관리자 - 예약 대시보드 (`/admin`)

| 요소 | 내용 |
|------|------|
| 통계 카드 | 오늘 예약 / 이번 주 / 전체 / 취소 |
| 검색·필터 | 이름·전화번호, 트레이너·날짜 필터 |
| 예약 테이블 | 예약자·전화·트레이너·날짜·시간·PT유형·상태 |
| 상태 변경 | BOOKED / COMPLETED / CAD / NO_SHOW 즉시 저장 |

### 7.8 관리자 - 회원 관리 (`/admin/members`)

| 탭 | 내용 |

## 8. 기능 명세서

### 8.1 회원 인증 (전화번호 + 비밀번호)

| 항목 | 내용 |
|------|------|
| 회원가입 | `POST /stores/:storeId/member/register` |
| 로그인 | `POST /stores/:storeId/member/login` |
| 토큰 검증 | `GET /stores/:storeId/member/me` |
| 비밀번호 해싱 | Node.js `crypto.scrypt` + 16바이트 랜덤 salt |
| JWT 유효기간 | 30일, role: 'member' |
| 저장 | `memberToken` localStorage |

### 8.2 전화번호 기반 회원 조회 / 생성 (Lookup)

| 항목 | 내용 |
|------|------|
| 용도 | 예약 폼에서 비로그인 예약 시 사용 |
| 처리 | GSI-1 (PhoneIndex) 전화번호 조회 |
| 기존 회원 | userId, name 반환; lastVisitAt 갱신 |
| 신규 | 이름 입력 → USER 레코드 + STORE-USER 레코드 생성 |

### 8.3 예약 생성

| 항목 | 내용 |
|------|------|
| 입력 | userId, date, time, trainerId(optional), ptType, passId(optional) |
| 처리 | TransactWriteCommand — 원자적 저장 |
| 중복 방지 | ConditionExpression: attribute_not_exists(PK) |
| 알림 | 예약 성공 시 SESSION_REMINDER 자동 생성 |

### 8.4 패스 세션 차감

| 항목 | 내용 |
|------|------|
| 자동 차감 | 예약 생성 시 TransactWrite 내 원자적 처리 |
| 수동 차감 | 관리자 PATCH API |
| 알림 연동 | 잔여 0 → REPURCHASE_PROMO / 2회 이하 → LOW_SESSIONS |

### 8.5 감사 로그 (Audit Log)

| 항목 | 내용 |
|------|------|
| 대상 | 회원 프로필 수정, 패스 수정, 예약 상태 변경 |
| 저장 | `PK: AUDIT#{entityType}#{entityId}`, `SK: {timestamp}#{uuid}` |
| 내용 | before/after 값, 수정자, 수정 사유, 일시 |
| 조회 | `GET /api/v2/audit/:entityType/:entityId` |

### 8.6 홈페이지 콘텐츠 관리 (CMS)

| 항목 | 저장 위치 | 내용 |
|------|----------|------|
| 로고 | `store.logo.{image, text}` | 헤더 로고 이미지 또는 텍스트 |
| 히어로 | `store.hero.{image, title, subtitle}` | 메인 배너 |
| 소개 섹션 | `store.aboutSections[0~1]` | 이미지·타이틀·본문 |
| 프로그램 카드 | `store.programs[]` | 최대 6개, title·desc·image |

### 8.7 결제 및 패스 자동 발급

| 단계 | 내용 |
