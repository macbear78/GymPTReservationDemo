## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 서비스명 | PT 예약 시스템 |
| 서비스 유형 | 헬스장 PT 온라인 예약 웹 서비스 |
| 플랫폼 | 웹 (PC / 모바일 반응형) |
| 목표 오픈일 | 협의 |

### 한 줄 요약

> 헬스장 회원이 트레이너를 선택하고 원하는 날짜·시간에 PT를 직접 예약할 수 있는 웹 서비스

---

## 2. 서비스 배경 및 목적

### 배경

- 기존 PT 예약은 전화·카카오톡 등 수기 방식으로 운영되어 시간 외 예약 접수가 불가능하고 직원 응대 부담이 큼
- 예약 현황을 엑셀이나 메모로 관리해 누락·중복 예약이 발생하는 경우가 있음
- 회원 입장에서 가능한 시간대를 실시간으로 확인하기 어려움

### 목적

- 24시간 온라인 예약 접수로 회원 편의성 향상
- 예약 중복 방지 및 현황 자동 집계로 운영 효율화
- 트레이너별 스케줄 관리 기반 마련

---

## 3. 타겟 사용자

### 일반 회원 (End User)

- 헬스장 등록 회원
- 모바일·PC에서 간편하게 예약하고 싶은 사용자
- 회원가입 없이 이름·전화번호만으로 이용 가능

### 관리자 (Admin)

- 헬스장 직원 및 원장
- 전체 예약 현황 조회·관리
- 트레이너 등록·관리

## 6. 사용자 흐름 (User Flow)

### 6.1 예약 흐름 (회원)

```
홈 접속
  └─▶ PT 예약하기 버튼 클릭
        └─▶ 트레이너 선택
              └─▶ 날짜 선택
                    └─▶ 시간 선택 (예약 가능 슬롯만 표시)
                          └─▶ PT 유형 선택 (30분 / 60분)
                                └─▶ 이름 · 전화번호 입력
                                      └─▶ 예약하기 클릭
                                            ├─▶ [성공] 예약 완료 화면
                                            └─▶ [실패] 오류 메시지 표시
```

### 6.2 내 예약 조회 흐름 (회원)

```
내 예약 메뉴 클릭
  └─▶ 전화번호 입력
        └─▶ 조회 클릭
              ├─▶ [결과 있음] 예약 목록 표시
              └─▶ [결과 없음] "예약 내역이 없습니다" 안내
```

### 6.3 예약 관리 흐름 (관리자)

```
관리자 페이지 접속 (/admin)
  └─▶ 전체 예약 목록 자동 로드
        ├─▶ 검색 · 필터 → 조건에 맞는 예약만 표시
        ├─▶ 상태 드롭다운 변경 → 즉시 저장
        └─▶ 취소 버튼 클릭
              └─▶ 확인 팝업
                    ├─▶ [확인] 예약 삭제 → 목록 새로고침
                    └─▶ [취소] 닫기
```

### 6.4 트레이너 등록 흐름 (관리자)

```
트레이너 등록 페이지 접속 (/admin/trainers)
  └─▶ 정보 입력 (ID, 이름, 전문분야, 경력, 가격)
        └─▶ 이미지 파일 선택 → 업로드 버튼 클릭
              └─▶ 이미지 S3 업로드 완료
                    └─▶ 트레이너 등록 버튼 클릭
                          ├─▶ [성공] 등록 완료 메시지
                          └─▶ [실패 - ID 중복] 오류 메시지
```

## 7. 페이지별 기능 정의

### 7.1 홈 (`/`)

| 섹션 | 내용 |
|------|------|
| 히어로 배너 | 헬스장 대표 이미지 + 슬로건 + CTA 버튼 (PT 예약하기 / 무료 상담) |
| 헬스장 소개 | 소개 문구 + 시설 이미지 |
| 트레이너 미리보기 | 트레이너 카드 3~4개 + 전체 보기 링크 |
| PT 프로그램 소개 | 근력 / 다이어트 / 재활 등 프로그램 유형 안내 |
| 시간표 | 운영 시간대 안내 |
| 연락처 / 오시는 길 | 주소, 전화번호, 카카오 채널 링크 |

**인터랙션**
- "PT 예약하기" → `/reserve` 이동
- "무료 상담" → 페이지 내 연락처 섹션으로 스크롤
- 트레이너 카드 클릭 → 트레이너 상세 페이지 이동

---

### 7.2 트레이너 목록 (`/trainers`)

| 요소 | 내용 |
|------|------|
| 트레이너 카드 | 프로필 이미지, 이름, 전문 분야, 경력, 평점 |
| 정렬 | 기본 등록 순 |

**인터랙션**
- 카드 클릭 → `/trainers/:id` 이동

---

### 7.3 트레이너 상세 (`/trainers/:id`)

| 요소 | 내용 |
|------|------|
| 기본 정보 | 이름, 전문 분야, 경력, 평점, 리뷰 수 |
| 소개 | 상세 소개 문구 |
| 전문 분야 태그 | 트레이닝 영역 목록 |
| 자격증 | 보유 자격증 목록 |
| 프로그램 | 제공 프로그램 목록 |
| 가격 | 시간당 가격 |
| CTA 버튼 | "이 트레이너로 예약하기" |

**인터랙션**
- "이 트레이너로 예약하기" → `/reserve`로 이동 (해당 트레이너 선택 상태)

---

### 7.4 PT 예약 (`/reserve`)

| 단계 | 요소 | 비고 |
|------|------|------|
| 1 | 트레이너 선택 | 등록된 트레이너 목록 표시 |
| 2 | 날짜 선택 | 달력 UI |
| 3 | 시간 선택 | 가능 슬롯만 활성화 (불가 슬롯 비활성) |
| 4 | PT 유형 선택 | 30분 / 60분 |
| 5 | 예약자 정보 | 이름, 전화번호 입력 |
| 6 | 예약 완료 | 예약하기 버튼 → 완료 화면 이동 |

**운영 시간 슬롯**

| 슬롯 | 시간 |
|------|------|
| 1 | 06:00 |
| 2 | 07:00 |
| 3 | 08:00 |
| 4 | 09:00 |
| 5 | 18:00 |
| 6 | 19:00 |
| 7 | 20:00 |

**유효성 검사**
- 모든 항목 필수 입력
- 이미 예약된 트레이너·날짜·시간 조합은 선택 불가
- 전화번호: 숫자 외 문자(공백, 하이픈) 자동 제거 후 처리

---

### 7.5 예약 완료 (`/complete`)

| 요소 | 내용 |
|------|------|
| 완료 메시지 | 예약 확정 안내 |
| 예약 요약 | 트레이너, 날짜, 시간, PT유형, 이름, 전화번호 |
| 버튼 | "홈으로 돌아가기" |

---

### 7.6 내 예약 조회 (`/my-reservations`)

| 요소 | 내용 |
|------|------|
| 전화번호 입력 | 숫자 입력 + 조회 버튼 |
| 예약 목록 | 날짜·시간·트레이너·PT유형·상태 표시 |
| 빈 상태 | "예약 내역이 없습니다" 안내 |

---

### 7.7 관리자 - 예약 관리 (`/admin`)

| 요소 | 내용 |
|------|------|
| 통계 카드 | 오늘 예약 수 / 이번 주 예약 수 / 전체 예약 수 / 취소 수 |
| 검색 | 이름 검색, 전화번호 검색 |
| 필터 | 트레이너 선택, 날짜 선택 |
| 예약 테이블 | 예약자·전화번호·트레이너·날짜·시간·PT유형·상태·취소 버튼 |
| 상태 변경 | 드롭다운으로 즉시 변경 저장 |
| 취소 처리 | 취소 버튼 → 확인 팝업 → 삭제 |

**예약 상태값**

| 값 | 표시 | 설명 |
|----|------|------|
| Confirmed | 확정 | 예약 접수 완료 |
| Completed | 완료 | PT 수업 완료 |
| Cancelled | 취소 | 예약 취소 |
| No-show | 노쇼 | 미방문 |

---

### 7.8 관리자 - 트레이너 등록 (`/admin/trainers`)

| 요소 | 내용 | 필수 |
|------|------|------|
| 트레이너 ID | 영문·숫자·_·- 조합, 고유값 | ✅ |
| 이름 | 표시될 트레이너 이름 | ✅ |
| 전문 분야 | 예: 근력 트레이닝 | ✅ |
| 경력 | 예: 5년 경력 | ✅ |
| 가격(원/시간) | 숫자 입력 | ✅ |
| 프로필 이미지 | 파일 선택 → 이미지 업로드 버튼 → S3 저장 | ✅ |

**유효성 검사**
- ID 중복 시 오류 메시지 표시
- 이미지 업로드 완료 전 등록 버튼 비활성화
- 이미지 파일만 선택 가능 (image/* MIME type)

## 9. 비기능 요구사항

### 9.1 반응형 디자인

- PC (1280px 이상), 태블릿 (768px~1279px), 모바일 (767px 이하) 모두 지원
- 모바일 우선(Mobile-First) 설계

### 9.2 성능

- 페이지 초기 로딩 3초 이내 (일반 네트워크 기준)
- 예약 생성 API 응답 3초 이내

### 9.3 브라우저 호환성

| 브라우저 | 지원 버전 |
|----------|----------|
| Chrome | 최신 2개 버전 |
| Safari | 최신 2개 버전 |
| Edge | 최신 2개 버전 |
| Firefox | 최신 2개 버전 |

### 9.4 보안 (현재 미구현, 추후 적용 권장)

- 관리자 페이지 접근 인증 (로그인)
- CORS 허용 도메인 제한
- 입력값 서버 측 유효성 검사 강화

## 13. 용어 정의

| 용어 | 설명 |
|------|------|
| PT | Personal Training. 1:1 개인 트레이닝 |
| 시간 슬롯 | 예약 가능한 특정 시간대 단위 (예: 09:00) |
| Confirmed | 예약 확정 상태. 회원이 예약을 완료하면 자동으로 부여되는 초기 상태 |
| Completed | 수업이 실제로 진행 완료된 상태 |
| Cancelled | 예약이 취소된 상태 |
| No-show | 회원이 예약 시간에 나타나지 않은 상태 |
| Presigned URL | S3에 파일을 직접 업로드할 수 있도록 서버가 일시적으로 발급하는 인증 URL |
| 관리자 | 헬스장 직원 또는 원장으로, `/admin` 경로에 접근해 예약을 관리하는 사용자 |
